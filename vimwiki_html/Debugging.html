GDB - GNU Project Debugger

To start

Compile program with -g flag then run: 

gdb ./a.out

To begin the program execution type 

run

or 

r

Then it will run the program and say if it exited normally or not. Then to set a breakpoint at a function named do_something, type: 

break do_something

Then type 

r

for run (or could do full run command) to run the program to that break point. It will tell the inputs to the function. Type: 

finish

or 

fin

to finish the specific function and return the output. Then to continue, type 

c

and it will continue to the next break point. Then type 

fin

to finish the next function call again.

To finish the program in the middle, type 

kill

Then you can check what breakpoints you have by typing 

info breakpoints

or 

i b

And you can delete Num 1 by typing (where Num lists the number of the breakpoint set) 

delete 1

You can also set conditional breakpoints by typing 

breakpoint do_something if x < 0

Which will only break at the function call if the x is less than 0.

gdb breakpoint commands can be seen here: https://sourceware.org/gdb/onlinedocs/gdb/Break-Commands.html

To print out a variable named i after you have hit a breakpoint, type: 

print i

To break at line 45 of file compile.cpp type: 

break compile.cpp:45

Also, to step through your program line by line, you can type (by default goes into functions... so you get constructors and things) 

s

and then just enter to repeat the step call.

Or to just go to the next line, and not go into the functions type 

n

To see where you are at in the command, you can type 

bt

to print out the backtrace of where you are. And it will tell you where you are by line, and by the frame indicated on the left by the # symbol.

Also, to see things further up you can specify the frame to get more info with 

f 1

to specify looking into frame 1

You can also set a variable to watch, to see if it gets written to by anything and stop the program. You can do this by typing 

watch &my_war;

To quit gdb, type 

quit

or 

q

To run GDB on an MPI parallel code

You can use a software called TotalView, or you can run it using GDB and terminator with the following command 

mpirun -n 2 terminator -x gdb ./a.out

This will open up to terminator terminals of gdb, then you just hit run in each one to get both processes running at the same time. MPI will wait until the MPI_INIT call is run on all processes.

Valgrind - memory management

To run, you just type 

valgrind ./a.out

Runs for a long time, but will list a summary at the end of the memory usage, and the leaks! Also lists uninitialized and can use flags such as 

--leak-check=full

To list the full leak summary output Or you can use 

--track-origins=yes

to see where the uninitialized variables were lost.
