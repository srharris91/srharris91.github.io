Information

This code will compute the Parabolic Stability Equation (PSE) for various flows. This will read in the Orr-Sommerfeld solution from a python script, and will march it forward in \(x\). If we march the parallel version of the PSE, the solution procedure would be defined below.

Solution Procedure

Some validation was conducted part of an instabilities project in Fall 2018, this report can be found <a href="ME452B_Instabilities_Project_Report.pdf target="_blank"><b>here</b></a> Additionally, the new nonlinear solution procedure was defined in more detail <a href="PSE_and_base_flow_derivation.pdf" target="_blank">here

\(\mathbf{A_{OSS}}\) would be different for the parallel PSE equation. Additionally, \(\mathbf{b_i}\) would also be different for the non-linear terms. (These are works in progress) All of the code is contained in the namespace PSE.

The PSE equation is defined here

\[ A \hat{q}_{n,m} + B \frac{\partial \hat{q}_{n,m}}{\partial y} + C \frac{\partial^2 \hat{q}_{n,m}}{\partial y^2} + D \frac{\partial \hat{q}_{n,m}}{\partial x} = F \]

Petsc Compile Info

Make sure to compile petsc with options

* ./configure --with-scalar-type=complex --with-precision=double
*

or with

* python2 './configure' '--with-scalar-type=complex' '--with-precision=double' '--download-mumps' '--download-scalapack' '--download-parmetis' '--download-metis' '--download-ptscotch'
*

or with the following to make the optimized version (instead of the default debug)

* python2 './configure' '--with-scalar-type=complex' '--with-precision=double' '--download-mumps' '--download-scalapack' '--download-parmetis' '--download-metis' '--download-ptscotch' '--with-cc=mpicc' '--with-cxx=mpicxx' '--with-fc=mpif90' '--with-debugging=0' 'COPTFLAGS='-O3 -march=native -mtune=native'' 'CXXOPTFLAGS='-O3 -march=native -mtune=native'' 'FOPTFLAGS='-O3 -march=native -mtune=native''
*

and follow the steps outlined in the output of this to obtain the MUMPS installation for LU decomposition in parallel. You may want to take note of the PETSC_DIR and PETSC_ARCH variables that are set, so you can use them in the SLEPc compilation

Also, to compile the slepc code, you may need to run the following command after setting the proper PETSC_DIR and PETSC_ARCH variables. Note: these variables are indicated while compiling PETSc

* python2 './configure'
*
